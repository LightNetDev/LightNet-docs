It is helpful to understand how your site is being used to improve the experience and measure the impact. Analytics
helps getting insights into what is happening on your site. They can work without compromising the privacy of your site's visitors. 
We recommend using 
[plausible](https://plausible.io) as an analytics solution. But you can also use other 
analytic providers with LightNet

## Setting up plausible tracking

With plausible](https://plausible.io) you can capture what is going on while
still protecting the privacy of your users: no storing of ip addresses, no cookies, no consent banners.
Here is how to set it up:

1. Create plausible account (plausible is paid with money not with your user's data)
2. In Plausible add new Website, select "NPM installation"
3. In LightNet add plausible script with `npm astro add @lightnet/plausible-analytics`. Confirm questions. This will add `@lightnet/plausible-analytics` dependency to your project and add it to your astro integration in astro.config.mjs.
4. ⚠️ In astro.config.mjs make sure Astro's `site` config is set to your site's actual url e.g. "https://your-site.com". The analytics depend on this setting.
4. Deploy your changes
5. In plausible verify your changes 

## Setting up other tracking providers

You can use any tracking provider by injecting the script with LightNet's [headComponent config option](/build/configuration/reference/#headcomponent).

1. in your LightNet project add a new Component "src/CustomHead.astro"
2. in your LightNet configuration in astro.config.mjs set `headComponent` to "src/CustomHead.astro".
3. In CustomHead.astro add your analytics script for example this is how to setup tracking with fathom analytics

```astro title="CustomHead.astro"
---
---
<script
  src="https://cdn.usefathom.com/script.js"
  data-site="YOUR-SITE-ID"
  defer>
</script>
```
