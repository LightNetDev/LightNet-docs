title: Markdown support
---

This page specifies Markdown support in LightNet.

## Text-based pages

Text-based pages may use any Markdown feature. See [Create an about page](/build/pages/about-page).

## Media item description text

The [description](/content/media-items/#description) text in Media Items supports only these Markdown features: 
- Bold
- Italics
- Underline
- Paragraphs
- Block quotes
- Headings
- Unordered lists
- Numbered lists
- Hyperlinks
- Quotation marks
- Special characters 

The example code below demonstrates how to declare a description.
Inside the Media Item JSON file the Markdown must be on a single line enclosed in double quotes.

``` markdown
"description": "This is a paragraph.\n\nThis is a second paragraph."
```

The examples below show the syntax for each supported Markdown feature.

### No Markdown
It is not a requirement that the description text contain anything other than basic text.

``` markdown
This is a plain text example.
```

### Bold
``` markdown
This is **bold** text.
```

Do not use underscores `__text__`. This is not supported.

### Italics
``` markdown
This is *italicized* text.
```

### Underline
Use the HTML underline tag.
``` markdown
This is <u>underlined</u> text.
```

This is the only supported HTML tag. See also [Other Markdown is not supported](#other-markdown-is-not-supported) below.

### Paragraphs
Use two newlines `\n\n` to create a new paragraph.

``` markdown
It begins.\n\nIt goes on and on.\n\nThen it ends.
```

Soft line breaks are not supported. Do not use two spaces followed by a newline. Do not use three backslashes followed by the character n. This is not supported.

### Block quotes
Use greater-than sign to start a block quote.

``` markdown
Some text.\n\n> This is a famous quotation.\n\nSome more text.
```

### Headings
Use hash for Heading 1 through Heading 6.
Example for Heading 2 using two hashes:
``` markdown
## A Sub-heading
```

### Unordered lists

``` markdown
This book offers:\n\n* Tips\n* Advice\n* Encouragement\n\nIt's great.
```

#### Nested unordered lists
Use two spaces to indent nested items.

``` markdown
This book offers:\n\n* Tips\n  * Advice\n* Encouragement\n\nIt's great.
```

Do not use four spaces to indent nested items. This is not supported.

### Numbered lists
``` markdown
This book offers:\n\n1. Tips\n2. Advice\n3. Encouragement\n\nIt's great.
```

#### Nested numbered lists
Use two spaces to indent nested items.

``` markdown
Steps:\n\n1. Start\n   1. Continue\n2. More to do\n\nDone.
```

Do not use four spaces to indent nested items. This is not supported.

### Hyperlinks
Hyperlinks are composed of a URL and Anchor text.
The URL is enclosed in parentheses.
The Anchor text, which is displayed to the user, is enclosed in square brackets.

``` markdown
Read: [John 3:16](https://www.bible.com/bible/111/jhn.3.16)
```

Optionally provide a Link title to provide supplemental information to the user about the URL.
This is displayed to the user as a tooltip for users on personal computers. 
It is not visible to users on mobile phones.
The Link title is enclosed in escaped double quotes within the parenthesis after the URL and a space character. 

``` markdown
Read: [John 3:16](https://www.bible.com/bible/111/jhn.3.16 \"Opens the NIV translation on Bible.com\")
```

### Quotation marks
Because the description text is enclosed in double quotes within the Media Item JSON file, double quotes to be displayed to the user must be escaped with backslash.

``` markdown
God said, \"Let there be light\".
```

Single quotes and smart quotes do not need to be escaped.

### Special characters
The backslash character must be escaped with another backslash in the description text in the Media Item JSON file.

``` markdown
This is a backslash \\ in the text.
```

These following special characters must be escaped with two backslashes in description text in the Media Item JSON file:
- ampersand followed by a letter or number `&`
- asterisk `*`
- backtick `` ` ``
- less-than sign `<`
- opening square bracket `[`
- opening round bracket `(`

``` markdown
To make a sad face, type a round bracket \\( followed by a colon :
```

### Other Markdown is not supported
Do not use other Markdown in the description text such as HTML tags (except for underline), soft line breaks, images, horizontal rules, code blocks, tables, or task lists.
