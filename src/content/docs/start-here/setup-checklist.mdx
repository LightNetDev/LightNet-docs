Most new media library deployments in LightNet follow a similar setup process. 
To make deployments easier and more consistent, we maintain a checklist of the tasks that are typically 
completed when launching a new media library.

You can use this checklist as a guide for your own deployment.

## How to use the checklist

The checklist defines three roles involved in the setup process. 
Depending on the size of your team, these roles may be fulfilled by three different people or by a single person.

* **Ministry**: The primary contact at the ministry or organization. 
    This person is responsible for project decisions and has the authority to approve purchases and subscriptions.
* **Site administrator (site admin)**: The developer responsible for configuring and deploying the media library. 
    Since you are reading the developer documentation, this is likely your role.
* **Content administrator (content admin)**: The person or team responsible for preparing and uploading content to the media library.

Not every task will apply to every ministry or organization. If an item is not relevant, 
either remove it or mark it as completed by striking it through using Markdown syntax (`~~text~~`).

We recommend copying the checklist into a GitHub issue so that progress can be tracked and tasks can be checked 
off throughout the setup process.
