Use language entries to define metadata for the languages available on your LightNet site.
Each file in `src/content/languages` describes one site language and is referenced from the `siteLanguages` configuration in `astro.config.mjs`.
Language entries are used for display information such as the names shown in language selectors.

Here is an example of a language entry:

```json title="src/content/languages/en.json"
{
  "code": "en",
  "label": {
    "en": "English",
    "de": "Englisch"
  }
}
```

<Aside>
Language entries complement the `siteLanguages` configuration in `astro.config.mjs`.
Use `siteLanguages` to configure which locales your site supports, and `src/content/languages` to define how those languages are displayed.
</Aside>

## Reference

A language entry is a JSON file that describes one site language.
The identifier is the filename without the `.json` suffix and should be a BCP-47 language code such as `en`, `de`, or `ar`.
The `code` value must match that filename.

### code

type: `string` \
example: `"en"` \
required: `true`

The BCP-47 language code for this language entry.
It must match the file name without the `.json` suffix.

### label

type: `object` \
example: `{ "en": "English", "de": "Englisch" }` \
required: `true`

The display name of the language used in the UI, for example in language selectors.
Provide a locale map keyed by your configured site languages.
The default site locale is required, while other configured site locales are optional.

For details on configuring site locales, see the [Internationalization Guide](/build/i18n/fundamentals) and [Add a language](/build/i18n/add-language).
